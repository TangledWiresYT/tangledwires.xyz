DeathMsgEdit

DeathMsgEdit lets you edit the death messages on your server.

Commands:

  • /deathmsgedit reload - Reload the config

    • How to use:

    When you start the server with the plugin installed for the first time, it will generate an example config, which you can edit to edit or add more death messages.

    If there are multiple entries for the same death cause, a random one will be selected each time.

    "{victim}" will be replaced with the name of the player that was killed.

    "{attacker}" will be replaced with the name of the player/mob that killed the player.

    If you have PlaceholderAPI installed, you can also use placeholders in death messages.

    You can also use colour codes to change how the text looks.

    messages:
    ENTITY_ATTACK:
    - 'Ouch! {victim} was killed by {attacker}!'
    - '{victim} isn't very good at Minecraft, they were just killed by {attacker}'

    This plugin uses bStats for analytics, the data collected is completely anonymous. If you would like to disable bStats, you can do so in the bStats config file.

    From the bStats website:

    "bStats does not collect any personal data. Most of the collected data is information about the plugin's server like player count, online mode, Minecraft version, Java version, and more. All data gets sent and stored completely anonymously. Server IPs are only stored for ratelimiting and not linked to the data. All data that is collected is publicly visible. If you are interested in what data is collected for your Minecraft server, take a look at the pages for the plugins on your server. Please note that plugin authors can also send data that is customized for their plugin."
